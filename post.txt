

I've been pretty happy with KitchenOwl, but the lack of ease of getting recipes into the service that didn't come from a website has held me back from full adoption. In the process of using Claude Code and having a done a few smaller projects, I thought I'd try something substantially more ambitious. Why not? Claude makes it super easy to create the initial product then iterate to a viable solution.

So, I had Claude create ko-pellet: a multi-source import for recipes into KitchenOwl. You can find the project here: https://github.com/Tiscin/ko-pellet . The import methods are: URL, image with mobile camera support, and text paste. The image and text paste import options are AI-assisted. If an API-key is not provided, Tesseract OCR is utilized.

This is my first publication of any kind, so would love any feedback ya'll have.

Important note: for something so simple (it's just recipes right?) access is secured behind OIDC login or forward auth via Authelia, Authentik, or similar. There is currently no support for a local account. There are a few reasons for this.

    I wanted to protect my valuable Anthropic and OpenAI API keys.

    I believe in zero-trust and defense-in-depth. Yeah, probably overkill for a project like this, but I have mechanisms and services in place and wanted to leverage that security.

    I didn't want yet another user/pass to remember.

    Because, well, it's just friggen cool.

Anyway, check out the project, especially if you're a KitchenOwl user. I hope you enjoy it as much as I know I'm going to.
